1. Infrastructure Architecture

            
The KoTH CTF platform consists of several Docker components running on a single server:

            

                
ContainerPortFunction

                
                    
koth-scoreboard:8000FastAPI — Scoreboard, API, Dashboard

                    
koth-scorebot:8080Go — SLA checker & scoring engine

                    
koth-nginx:80Reverse proxy — serve all traffic

                    
koth-postgres:5432PostgreSQL 16 — main database

                    
koth-redis:6379Redis 7 — cache & WebSocket pub/sub

                
            

            
Hill servers are separate machines running koth-agent. Scorebot performs SLA checks on every hill each tick.


            
            
2. Login Organizer Console

            
1
Open browser, navigate to https://YOUR_DOMAIN/org

            
2
Enter the Admin Token (defined in .env as API_ADMIN_TOKEN)

            
3
Click Login — you will be taken to the Organizer Dashboard

            
After logging in, use the sidebar menu on the left to navigate to all features.


            
            
3. Setup Event Details

            
Set up event information that will be displayed as the participant landing page after login.

            
1
Click 📋 Event Details in the sidebar

            
2
Fill in the form: Event Name, Subtitle, Organizer, Contact, Date, Location, Description, and Rules

            
3
Check the Live Preview in the right panel to verify the display

            
4
Click the 💾 Save Event Details button

            
Verify: Open the participant dashboard — event information will appear in the top banner of the Home page.


            
            
4. Manage Team Categories

            
Categories are used to group teams (e.g.: Military, Civilian, Student, etc.).

            
1
Click 🏷️ Categories in the sidebar

            
2
To add a category: fill in Category ID (slug, lowercase no spaces, e.g.: military) and Display Label (e.g.: Military)

            
3
Click ➕ Add — new category appears in the table

            
4
To edit a label: directly modify the Label column in the table

            
5
To delete: click the 🗑️ button (default category cannot be deleted)

            
6
Click 💾 Save Categories to save changes

            
Verify: Open the registration page — the category dropdown will show the categories you created.


            
            
5. Game Configuration

            
Set game parameters before the event starts.

            
1
Click ⚙️ Config in the sidebar

            
2
You will see the configuration table. Important parameters:

            

                
KeyDefaultDescription

                
                    
tick_interval60Scoring interval in seconds

                    
grace_period300Initial grace period without scoring (5 minutes)

                    
game_duration21600Total game duration in seconds (6 hours)

                    
freeze_before_end1800Freeze scoreboard X seconds before end

                    
first_blood_bonus50First blood bonus per hill

                    
defense_streak_bonus5Defense streak bonus per tick

                    
base_points10Base points per tick per hill

                
            

            
3
Click the ✏️ Edit button on the row you want to change

            
4
Enter the new value, then click 💾 Save

            
⚠️ Change configuration before the game starts. Changing during gameplay may cause inconsistent scores.


            
            
6. Add & Manage Hills

            
1
Click 🏔️ Hills in the sidebar

            
2
To add a new hill, fill in the form at the top:
                
    
                    
  • Name — Hill name (e.g.: "Web Server")
    • 
                    
  • IP Address — Hill VPC IP
    • 
                    
  • SSH Port — Hill SSH port
    • 
                    
  • SLA Check — SLA check type (http, tcp)
    • 
                    
  • SLA Port — Port checked for SLA
    • 
                    
  • Base Points — Points per tick
    • 
                    
  • Multiplier — Score multiplier (1.0 normal, 1.5 for pivot)
    • 
                

            

            
3
Click Create Hill

            
4
To disable a hill (e.g. technical issues): click the Toggle button on the hill row

            
Verify: Check 🔍 Connection Status to ensure all hills are connected and SLA OK.

8. Participant Registration

            
Self-registration allows participants to register themselves without needing to be created by the organizer.

            
1
Click 📝 Registration in the sidebar

            
2
Click the Toggle button to enable registration (status changes to ON)

            
3
Share the registration link with participants: https://YOUR_DOMAIN/register

            
4
Participants need to enter the Registration Code (defined in .env as REGISTRATION_CODE)

            
5
After all participants have registered, click Toggle again to close registration (OFF)

            
⚠️ Share the registration code during briefing, not before. Close registration before the game starts.

            
Verify: Check 👥 Teams — registered teams will appear in the table.


            
            
9. Run Game (Start / Pause / Stop)

            
1
Click 🎮 Game Control in the sidebar

            
2
Current game status is displayed at the top. Initial: idle

            
▶️ Start Game

            
3
Click the ▶️ Start Game button

            
4
Game enters Grace Period (5 minutes) — participants can login but no scoring yet

            
5
After grace period, status changes to running and the first tick begins

            
⏸️ Pause / ▶️ Resume

            
•
Click ⏸️ Pause to temporarily stop scoring (e.g. technical issues)

            
•
Click ▶️ Resume to continue the game

            
⏹️ Stop Game

            
•
Click ⏹️ Stop to end the game

            
⚠️ STOP is FINAL! The game cannot be resumed after stopping. Make sure this is indeed the end of the match.

            
Status Game

            

                
StatusDescription

                
                    
idleGame has not started

                    
grace_period5-minute grace period — no scoring

                    
runningGame running, scoring active each tick

                    
pausedGame paused, scoring stopped

                    
stoppedGame finished

                
            


            
            
10. Scoreboard Control

            
Freeze / Unfreeze

            
1
Click 🎮 Game Control in the sidebar

            
2
In the Scoreboard section, click 🧊 Freeze to freeze the public scoreboard display

            
3
Scoring continues behind the scenes — participants just can't see updates

            
4
After the game ends, click 🔥 Unfreeze to reveal final scores

            
Auto-freeze: Scoreboard automatically freezes 30 minutes before the game ends (configurable in Config: freeze_before_end).

17. Troubleshooting

            
Hill SLA Failed

            
1
Check 🔍 Connection Status — see which hill has errors

            
2
SSH to hill server, check services: docker ps, systemctl status koth-agent

            
3
If needed, temporarily disable hill: 🏔️ Hills → Toggle off

            
Scorebot Error

            
1
Check logs: docker logs --tail 50 koth-scorebot

            
2
Restart: docker restart koth-scorebot

            
Scoreboard App Error

            
1
Check logs: docker logs --tail 50 koth-scoreboard

            
2
Restart: docker restart koth-scoreboard

            
Game Reset (⚠️ DESTRUCTIVE!)

            
⚠️ EXTREME CAUTION: Reset deletes ALL scoring data. Teams remain but all points become 0.

            
1
Stop the game first via 🎮 Game Control → Stop

            
2
Use the API: POST /api/admin/game/reset with X-Admin-Token header

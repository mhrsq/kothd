📖 How to Connect — WireGuard

1

Install WireGuard

sudo apt update && sudo apt install -y wireguard

Windows: Download dari wireguard.com/install

2

Download & Import Config

Klik tombol "🛡️ Download WireGuard Config" di atas, lalu:

Linux:

sudo cp wg-team.conf /etc/wireguard/wg0.conf

Windows: Buka WireGuard GUI → Add Tunnel → pilih file wg-team.conf

3

Aktifkan Koneksi

sudo wg-quick up wg0

Windows: Klik Activate di WireGuard GUI

4

Verifikasi Koneksi

Cek status WireGuard:

sudo wg show

Scan VPN network untuk discovery:

nmap -sn 10.0.0.0/24

Scan port target yang ditemukan:

nmap -sT -Pn <target-ip>
5

Matikan VPN (jika perlu)

sudo wg-quick down wg0

📖 How to Connect — OpenVPN

1

Install OpenVPN

sudo apt update && sudo apt install -y openvpn

Windows: Download dari openvpn.net/community-downloads

2

Download & Connect

Klik tombol "🔑 Download OpenVPN Config" di atas, lalu:

Linux:

sudo openvpn --config team.ovpn

Tunggu sampai muncul: Initialization Sequence Completed

Windows: Buka OpenVPN GUI → Import file team.ovpn → Connect

3

Verifikasi Koneksi

Cek interface tunnel:

ip addr show tun0

Ping ke VPN network:

ping -c 3 <koth-server-ip>

⚠️ Troubleshooting

🔴 Tidak bisa ping?▶
Pastikan VPN aktif (sudo wg show), cek apakah firewall lokal memblokir, dan pastikan routing ke 10.0.0.0/24 melewati interface VPN.
🔴 DNS tidak resolve?▶
Gunakan IP langsung. Hills diakses via IP, bukan domain.
🔴 Koneksi lambat?▶
Cek MTU: sudo ip link set mtu 1280 dev wg0
🔴 Hill 3 & 4 unreachable?▶
Hills ini di belakang Pivot. Pastikan routing lewat pivot aktif. Anda mungkin perlu pivot melalui DMZ.

1. Apa itu King of the Hill?

King of the Hill (KoTH) adalah format kompetisi CTF di mana tim-tim berlomba untuk:

  1. 🔓 Meretas (Capture) server target ("Hill") melalui eksploitasi vulnerability
  2. 👑 Menjadi King dengan menulis nama tim ke file /root/king.txt
  3. 🛡️ Mempertahankan (Defend) akses agar tetap King
  4. 💰 Mengumpulkan Poin — setiap 60 detik, King mendapat poin

Semakin lama kamu memegang Hill, semakin banyak poin yang kamu dapat!

┌─────────────────────────────────────────────────────────┐
│   4 Hills ─── Exploit ─── Root Access ─── king.txt     │
│   Setiap 60 detik: Scorebot baca king.txt              │
│   King + SLA hidup = +10 poin × multiplier             │
│   Service mati = 0 poin (meskipun kamu King!)          │
│   First Blood (pertama capture) = +50 bonus            │
│   Defense Streak (3+ tick berturut) = +5 bonus/tick    │
│   Hills di belakang Pivot = 1.5× multiplier            │
│   Durasi: 6 jam | 30 menit terakhir = Scoreboard FREEZE│
└─────────────────────────────────────────────────────────┘

2. Cara Bermain: Capture & Defend

Step 1: Recon & Scanning

# Scan subnet untuk menemukan target hills
nmap -sn 10.0.0.0/24

# Setelah menemukan IP hill, scan port & services
nmap -sT -sV -Pn -p- <hill-ip>
gobuster dir -u http://<hill-ip> -w /usr/share/wordlists/dirb/common.txt

Step 2: Exploit & Get Root

sudo -l                         # Check sudo permissions
find / -perm -4000 2>/dev/null  # Find SUID binaries
cat /etc/crontab                # Check cron jobs

Step 3: Claim the Hill! 👑

echo "team-name-kamu" > /root/king.txt
⚠️ Nama harus PERSIS sama dengan Team Name yang terdaftar (case-sensitive). Cek di Dashboard → Info Team. File harus di /root/king.txt.

Step 4: Defend! 🛡️

# Ganti password root
passwd root

# Kill shell orang lain
w
ps aux | grep -E "ssh|bash|nc|python"
kill -9 <PID>

# Monitor king.txt
watch -n 5 cat /root/king.txt

# Protect king.txt
chattr +i /root/king.txt
⚠️ JANGAN MATIKAN SERVICE! Jika SLA mati, kamu mendapat 0 poin meskipun kamu King!

3. Mekanisme Scoring

Scoring Formula

Poin per Tick = Base Points (10) × Hill Multiplier × SLA Status
KomponenNilai
Base Points10 poin per tick per hill
Multiplier Hill 1 & 21.0× → 10 poin/tick
Multiplier Hill 3 & 41.5× → 15 poin/tick
SLA Down0 poin!

Bonus Poin

BonusPoinKondisi
🩸 First Blood+50Tim pertama capture hill (per hill, hanya sekali)
🛡️ Defense Streak+5/tickSetelah memegang hill ≥ 3 tick berturut-turut

Contoh Perhitungan (Hill 1, 10 tick)

Tick 1: 10 × 1.0 = 10 poin (+ 50 first blood jika pertama)
Tick 2: 10 × 1.0 = 10 poin
Tick 3: 10 × 1.0 + 5 = 15 poin (streak bonus mulai!)
Tick 4-10: 15 poin/tick
Total: 60 + 40 + 50 (FB) = 150 poin

SLA Check

HillSLA CheckYang Dicek
Hill 1HTTPResponse 200 OK dari port 80
Hill 2TCPKoneksi TCP ke port 9999
Hill 3HTTPResponse 200 OK dari port 8080
Hill 4TCPKoneksi TCP ke port 27017

Dual Verification

Sistem menggunakan 2 metode: SSH Check (scorebot SSH ke hill) dan Agent Check (agent di hill melapor ke server). Jika salah satu gagal, metode lain tetap bisa digunakan.

Scoreboard Freeze

30 menit terakhir (14:30-15:00), scoreboard di-freeze. Score tetap dihitung tapi posisi di leaderboard tidak update. Setelah game selesai, scoreboard akan di-unfreeze untuk reveal hasil akhir.

nc -lvnp 4444

# Reverse shell payload (di target)

# SSH brute-force

9. Strategi & Tips

🎯 Strategi Umum

  1. Scan ALL hills dulu sebelum mulai exploit
  2. Prioritaskan hill mudah untuk first blood (+50)
  3. Hill 3 & 4 (1.5×) lebih menguntungkan jangka panjang
  4. Defense streak (+5 setelah 3 tick) berharga — pertahankan!
  5. Jangan fokus 1 hill — hold minimal 2 hill sekaligus
  6. Pembagian tugas — 1-2 attack, 1-2 defend

🧠 Tips Pro

  • Automate king claim: while true; do echo "team" > /root/king.txt; sleep 10; done &
  • Setup SSH key setelah dapat root agar login tanpa password
  • Pasang alarm jika king.txt berubah dengan inotifywait
  • Cek SLA dari perspektif kamu: curl -s -o /dev/null -w "%{http_code}" http://<hill-ip>:<port>
  • Jangan ubah firewall yang memblokir scorebot!

10. Troubleshooting

Tidak Bisa Ping Hill

  1. Pastikan VPN aktif: sudo wg show
  2. Cek routing: ip route | grep 10.0
  3. Cek firewall lokal: sudo iptables -L -n
  4. Coba MTU lebih kecil: sudo ip link set mtu 1280 dev wg0

Hill 3 & 4 Unreachable

Hill 3 & 4 ada di belakang Pivot. Kamu perlu scan dan temukan sendiri bagaimana cara mengaksesnya. Hint: coba scan pivot host untuk menemukan port mapping.

King.txt Tidak Ke-detect

  1. Nama PERSIS sama (case-sensitive)
  2. Tidak boleh ada spasi/newline tambahan
  3. File harus di /root/king.txt
  4. Cek: cat -A /root/king.txt (seharusnya team-name$)

Service Mati (SLA Fail)

# Restart web service (Hill 1 contoh)
systemctl restart apache2
ss -tlnp | grep :80
curl -s -o /dev/null -w "%{http_code}" http://localhost:80
